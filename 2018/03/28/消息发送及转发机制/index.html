一、消息发送机制

需要了解OC里面消息的发送机制，你的先了解对象的结构，这里不花太多时间去讲解类的结构相关知识，你的了解它有那么一个结构体，有一个cached对象，专门用来缓存近期使用过的方法，还有一个method对象，存储这该对象的所有方法列表。知道有这么一回事后，我们就好讲解消息发送的过程了。
对象调用的方法最终都是通过消息发送的，转换成obj_msgSend(id,sel)函数。那么调用方法OC里面到底是怎么一个流程呢？

  1. 检测@selector是否需要忽略。像具有垃圾回收机制的Mac OX环境，就不会去理会retain release等函数。
  2. 检测target是否为nil.在OC里面是允许nil对象发送消息的，并且不会Crash掉，因为在此直接忽略掉了。
  3. 如果上面两个过程都通过了，则从cached中寻找是否有该消息。找到了就直接执行。
  4. 如果在cache中没有找到，那么就会去该对象的方法分发表。
  5. 如果该对象的方法分发表中没有找到则去父类继续找，一直找到NSObject。
  6. 如果一直找不到。那么消息发送的过程也就到此告一段落了。接下来就要进行消息转发。以至于程序不会那么容易Crash掉。
